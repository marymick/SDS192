Projects

Everything in this course builds up to the final group project, where there is only one learning goal: Engage in the data/science research pipeline in as faithful a manner as possible while maintaining a level suitable for novices. To get practice however there will be 3 mini-projects first.

1. Mini-project 1: Data visualization (Due 10/5 5pm)

  • Structure Near identical to as in Spring 2018, with a few differences in submission format and grading rubric
  • Learning Goal To make informative, appropriate, and compelling data graphics.
  • Technical Skills ggplot2 package
  • Readings ModernDive as chief readings, Chapters 2 & 3 of “Modern Data Science with R” as supplementary readings if you want more details.

Details

You will work with a partner to write a short blog post that contains (at least) one data graphic. Your goal is to tell us something interesting using a well-crafted, thoughtfully-prepared data graphic. One data graphic should suffice, but you may include more if you choose.

Your blog post should be short. We envision an introductory paragraph that explains your findings and provides some context to your data, the data graphic(s), and then a caption-like paragraph providing more detail about what to look for in the data graphic and how to interpret it. That is it. You will not earn more points by including more words or data graphics. What we are looking for is something that is insightful and well-crafted. As always, you should spend some time thinking about context, scale, color, etc.

Here are some examples of articles that are similar in spirit to yours. Most of these are much longer than yours will be, and may contain multiple graphics, but the idea is similar: use a good data graphic to tell us something we don’t already know.

  • How to Tell Someone’s Age When All You Know Is Her Name
  • A Better Way To Find The Best Flights And Avoid The Worst Airports
  • NYC Taxis and Uber
  • Data on people who went to ER for wall-punching

Data

You are free to use whatever data you want. However, the purpose of this exercise is to learn how to make good data graphics – not to wrangle data (don’t worry – we’re headed there next). So we don’t want you to spend much time wrangling data. There are perfectly good data sets available through R packages that are already well-curated:

  • nycflights13: data about flights leaving from the three major NYC airports in 2013
  • babynames: history of baby names from the Social Security Administration
  • Lahman: comprehensive historical archive of major league baseball data
  • fueleconomy: fuel economy data from the EPA, 1985–2015
  • fivethirtyeight: provides access to data sets that drive many articles on FiveThirtyEight. See a list of all datasets here.

To view all the datasets included in a package, the babynames dataset for example, run the following in the console:

# Remember, you need to always load/open the package first!
library(babynames)
data(package = "babynames")

Grading Rubric

There are 17 possible points for this mini-project.

Baseline

  • +1 for R code that generates the plot without errors and is reproducible and replicable. In other words, if you send the .R file to your collaborator, would they be able to recreate your work? See Problem set 2
  • +1 for describing the dataset
  • +1 for having a data graphic
  • +1 for including the code that generated the data graphic
  • +1 for describing the visual mapping (i.e. a key)

Average

  • +1 for including a catchy and/or engaging title
  • +1 for having at least 100 words and no more than 300 words
  • +1 for explaining in a single coherent sentence what we can learn from this graphic
  • +1 for explaining the choice of geometric mapping
  • Edited on 9/28: +1 for explaining the color mapping justifying/explaining the choice of color mapping, even if none. In other words, do not add color for sake of adding color (this is called chartjunk). Rather use color only if you feel it adds value to your plot.
  • +1 for explaining which variables are mapped to which visual cues, and why

Advanced

  • +2 blog post text provides context or background useful in interpreting the graphic
  • +0–4 WOW factor: awarded at the grader’s discretion for submissions that are exceptionally compelling
